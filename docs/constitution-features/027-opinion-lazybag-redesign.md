## Stage Report

### 1. Research: 114年憲判字第1號判決 — DONE

114年憲判字第1號是憲法法庭針對「國會職權修法案」（立法院職權行使法修正案）所做出的判決。此案涉及2024年立法院通過的多項擴權條文，被聲請違憲審查。

**判決涉及的核心爭議條文：**
- 總統國情報告（強制即問即答）
- 立法院聽證調查權（強制出席、罰鍰、移送偵辦）
- 人事同意權（反質詢、資格審查程序）
- 藐視國會罪（刑事處罰公務員）
- 國會調查權與司法/監察權的界線

**主要立場陣營：**
- 多數意見：認定多數條文違憲，侵害權力分立原則
- 協同意見：同意違憲結論但補充不同論理基礎
- 不同意見：認為部分或全部條文合憲，屬立法形成自由

### 2. Key argument dimensions — DONE

針對114年憲判字第1號，論證可歸類為以下核心維度：

| 維度 | 支持端 | 反對端 |
|------|--------|--------|
| 權力分立 | 嚴格權力分立（國會不能逾越） | 國會至上（民意代表應有廣泛職權） |
| 總統國情報告 | 總統有行政特權，不受即問即答 | 總統應向國會負責、接受質詢 |
| 調查權範圍 | 調查權應受嚴格限制（不得侵入司法/監察） | 國會應有完整調查權以監督行政 |
| 藐視國會罪 | 刑事處罰違反比例原則、侵害言論免責 | 需有強制力才能落實監督功能 |
| 人事同意程序 | 現行程序已足夠，新規定逾越憲法 | 國會應有實質審查與否決空間 |
| 立法程序正當性 | 修法過程有重大瑕疵（議事規則違反） | 多數決即為民主正當程序 |

### 3. Alternative visualization formats — DONE

#### 方案 A：爭議條文拆解卡片（Issue Breakdown Cards）

每張卡片代表一個爭議條文，卡片內以「正反對照」方式呈現：
- 上方：條文簡述（一句話說明這條在幹嘛）
- 中間分兩欄：「為什麼違憲」vs「為什麼合憲」
- 下方：最終判決結果 badge（違憲/合憲/部分違憲）
- 可展開：看完整論證摘要

**互動方式：** 使用者逐張瀏覽，或用 filter 篩選「違憲」「合憲」條文。手機上為垂直滾動卡片流。

**為何更好：** 以「條文」為單位（而非抽象維度），使用者一眼看出「立法院通過了什麼、法庭怎麼判」。正反對照讓讀者不需預設立場也能理解雙方論點。

#### 方案 B：判決結果流程圖（Decision Flowchart）

以流程圖/決策樹呈現整個判決的邏輯結構：
- 起點：「立法院通過了這些修法」
- 分支節點：每個條文 → 「法庭怎麼看？」
- 終點：每條路徑最終到達「違憲」或「合憲」的結論
- 在每個節點可展開看正反論證

**互動方式：** 使用者可點擊節點展開，或沿路徑追蹤。手機上轉為垂直 timeline/steps 呈現。

**為何更好：** 將判決呈現為一個「故事」而非資料集，使用者跟著判決邏輯走，自然理解結論。但缺點是流程圖在手機上較難呈現。

#### 方案 C：立場光譜比較表（Position Spectrum Table）

以表格 + 色彩光譜呈現：
- 列（rows）= 爭議條文
- 欄（columns）= 不同意見類型（多數/協同/不同意見）
- 每格用顏色 + 一句話摘要顯示該意見對該條文的立場
- 頂部有「懶人包摘要」：快速看全部結果

**互動方式：** hover 或 tap 某格看完整論證。可切換「簡易模式」（只看結果色塊）與「詳細模式」（看論證摘要）。

**為何更好：** 保留了原版「比較不同立場」的優點，但用表格取代散布圖，使用者不需理解座標軸。色彩直覺（綠=合憲、紅=違憲、黃=部分）降低認知負擔。

### 5. Content structure — DONE

以推薦方案 A（爭議條文拆解卡片）為基礎的內容結構：

```
頁面結構
├── Header（標題、判決編號、一句話背景說明）
├── TL;DR 摘要區
│   ├── 「這個判決在說什麼？」（2-3 句白話說明）
│   ├── 結果統計：N 條違憲 / N 條合憲 / N 條部分違憲
│   └── 關鍵字 tags（權力分立、調查權、藐視國會...）
├── 爭議條文卡片區（主要內容）
│   ├── 卡片 1: 總統國情報告（即問即答條款）
│   │   ├── 條文白話翻譯
│   │   ├── 正方論點（為何合憲）
│   │   ├── 反方論點（為何違憲）
│   │   └── 判決結果 badge
│   ├── 卡片 2: 聽證調查權
│   ├── 卡片 3: 藐視國會罪
│   ├── 卡片 4: 人事同意權
│   └── 卡片 5: 調查權界線
├── 整體判決脈絡區（可收合）
│   ├── 時間線：修法 → 聲請 → 暫時處分 → 判決
│   └── 各意見書類型說明
└── CrossTrackLinks（連結到軌道二的完整判決分析）
```

**資料需求：**

```typescript
interface RulingIssue {
  id: string;
  title: string;           // e.g., '總統國情報告'
  subtitle: string;        // e.g., '強制即問即答條款'
  articleRef: string;      // 條文編號
  plainExplanation: string; // 白話翻譯（一般人看得懂）
  verdict: 'unconstitutional' | 'constitutional' | 'partial';
  proArguments: ArgumentPoint[];  // 合憲論點
  conArguments: ArgumentPoint[];  // 違憲論點
  tags: string[];
}

interface ArgumentPoint {
  summary: string;         // 一句話摘要
  detail?: string;         // 展開後的完整論證
  sourceType: 'majority' | 'concurring' | 'dissenting';
}

interface RulingOverview {
  rulingRef: string;       // '114年憲判字第1號'
  tldr: string;            // 白話 TL;DR
  totalIssues: number;
  verdictBreakdown: { unconstitutional: number; constitutional: number; partial: number };
  timeline: TimelineEvent[];
  issues: RulingIssue[];
}
```

### 7. Recommendation — DONE

**推薦方案 A：爭議條文拆解卡片（Issue Breakdown Cards）**

理由：
1. **最直覺**：以「條文」為單位分析，一般人最熟悉「法律在規定什麼」的思維框架，不需先理解抽象維度。
2. **手機友善**：卡片天然適合垂直滾動，不需特殊 responsive 處理。
3. **與設計系統一致**：符合軌道二的 Editorial Card 風格，可復用 border/typography 規範。
4. **開發成本低**：不需 SVG/Canvas，純 HTML/CSS 組件即可。比散布圖簡單。
5. **正反對照設計**：讓使用者看到雙方論點而非被引導到某一方結論，符合專案「理解光譜而非評斷對錯」的宗旨。

補充建議：
- 可在卡片區上方加入「快速結果一覽」（小型光譜表），作為全覽入口
- 未來若有更多判決要加入，卡片式設計易於擴展（每個判決一組卡片）
- 原有的 OpinionTable 可降級為進階使用者的「完整資料檢視」，放在頁面底部可選顯示
