需要規劃：

## Stage Report

### 1. 判決重點摘要（供漫畫改編）

114年憲判字第1號是憲法法庭針對2024年立法院通過的「國會職權修法」所作的判決。核心爭議：立法院擴權是否逾越憲法分際。法庭逐條審查五大條文，多數宣告違憲或部分違憲。

**五大爭議條文：**
1. **總統國情報告（即問即答）** → 違憲。總統不對立法院負責，不得強制質詢。
2. **聽證調查權** → 部分違憲。調查權本身合憲，但強制民間人士出席、罰鍰規定違憲。
3. **藐視國會罪** → 違憲。構成要件模糊，違反法律明確性與比例原則。
4. **人事同意權** → 部分違憲。修法過程有重大議事程序瑕疵（未經實質討論即表決）。
5. **調查權界線** → 違憲。不得擴張至偵查中案件，違反司法獨立。

**核心訊息（漫畫要傳達的）：** 權力需要界線，立法院不能自己給自己無限權力——這就是憲法法庭存在的意義。

### 2. 漫畫分鏡腳本（10 格）

採「故事線」結構，以貓頭鷹法官（現有吉祥物）為敘事者，帶領讀者走過整個事件。

| 格數 | 畫面 | 文字（旁白/對白） |
|------|------|-------------------|
| **1. 封面格** | 貓頭鷹法官站在憲法法庭門口，手持「憲法」書，上方大標題 | 旁白：「國會可以自己決定自己有多大的權力嗎？」／標題：114年憲判字第1號 懶人包 |
| **2. 背景格** | 立法院議場場景（簡化），一群人舉手表決，文件飛舞 | 旁白：「2024年，立法院通過了一系列『國會改革』法案，大幅擴張立法院的權力。」 |
| **3. 爭議格** | 畫面分成五個小區塊，每塊用一個icon代表一項修法（麥克風＝國情報告、放大鏡＝聽證、手銬＝藐視國會罪、人形＝人事同意權、資料夾＝調查權） | 旁白：「一口氣修了五大項——但這些權力，憲法有給嗎？」 |
| **4. 釋憲格** | 民眾、行政院、在野黨分別從不同方向走向憲法法庭大門 | 旁白：「各方聲請釋憲。憲法法庭受理，開始逐條審查。」 |
| **5. 國情報告格** | 貓頭鷹法官搖頭，旁邊畫一個被麥克風纏住的總統形象（誇張化） | 貓頭鷹：「總統不對立法院負責，不能強迫他站在台上被『質詢』。」／判定標籤：❌ 違憲 |
| **6. 聽證＋藐視格** | 上半：放大鏡照向一般民眾，民眾驚慌；下半：手銬旁邊寫著模糊的法條文字 | 貓頭鷹：「調查可以，但不能隨便罰人民。『藐視國會罪』連什麼叫藐視都說不清楚。」／判定標籤：⚠️ 部分違憲 ＋ ❌ 違憲 |
| **7. 程序格** | 表決器快速按下的特寫，旁邊時鐘指針飛速轉動 | 貓頭鷹：「而且……這些法案在立法院表決時，根本沒有好好逐條討論！」／判定標籤：⚠️ 程序違憲 |
| **8. 調查權格** | 立法院的手伸向法院和檢察署的門，被一道屏障擋住 | 貓頭鷹：「立法院的調查權不能伸進法院正在辦的案子裡——這叫司法獨立。」／判定標籤：❌ 違憲 |
| **9. 總結格** | 貓頭鷹法官站在天秤前，天秤兩端分別是「國會權力」和「憲法界線」，天秤平衡 | 旁白：「權力分立，不是誰大誰小——是每個機關都有自己的界線，誰都不能越線。」 |
| **10. CTA格** | 貓頭鷹法官指向觀眾，下方有網站連結和QR code預留位置 | 貓頭鷹：「想看完整分析？來 Add C0urt 憲庭加好友！」／副文：addcourt.tw |

### 3. 角色設定

- **主角：貓頭鷹法官（現有吉祥物）**
  - 沿用網站既有設計：深棕色貓頭鷹、紫色法袍、學士帽、持「憲法」書
  - 漫畫中增加表情變化：嚴肅（講解時）、搖頭（違憲時）、比讚（總結時）
  - 風格：保持現有卡通風格的一致性，線條清晰、色彩飽和

- **配角（簡化剪影/象徵性角色）：**
  - 立法院議員群：以藍綠灰色剪影群體呈現，不具體指涉個人
  - 總統：西裝剪影＋國旗胸章，不畫具體臉孔
  - 一般民眾：簡單卡通人形，表達「被影響的人民」

- **風格基調：** 教育性漫畫，友善但不幼稚。類似「法律白話文運動」的圖文風格——資訊密度適中，每格都能獨立傳達一個概念。

### 4. 視覺規範

**色彩（沿用網站色彩系統）：**
- 主色：gray-800 (#1f2937) — 文字、邊框
- 強調色：accent-red (#D32F2F) — 違憲標記、重要警示
- 輔助色：amber (#F7C04A / amber-400) — 部分違憲、注意
- 背景色：pearl (#F6F8FA) — 整體背景
- 合憲綠：green-600 — 若有合憲標記時使用

**字體：**
- 標題/旁白：Noto Serif TC Bold（與網站 serif 一致）
- 對白氣泡：Noto Sans TC Regular
- 標籤/badge：Manrope Bold + 等寬字體風格（與網站 badge 一致）

**視覺元素：**
- 每格邊框：1px solid gray-200，圓角 rounded-sm（與網站卡片風格一致）
- 判定標籤使用網站既有的 badge 風格：bg-red-100 text-red-800（違憲）、bg-amber-100 text-amber-800（部分違憲）
- 每格之間以灰色連接線串聯（呼應 DecisionFlowchart 的 connector line 設計）

### 5. 格式規格

**A. 網頁內嵌版（Web Embed）**
- 格式：SVG（向量，可縮放）或高解析度 PNG
- 寬度：最大 768px（配合網站 max-w-3xl 容器）
- 高度：依內容自適應，每格約 200-250px 高
- 整體呈「垂直長條圖」排版，適合網頁滾動閱覽
- 備選方案：以 React 元件實作（可互動，與網站技術棧一致）

**B. IG Carousel 版**
- 格式：PNG（1080 x 1080px，72dpi，sRGB）
- 張數：10 張（每格一張，對應上方 10 格分鏡）
- 第 1 張為封面（含標題＋吉祥物），第 10 張為 CTA（含網站連結）
- 安全區域：上下左右各留 50px，避免被 IG UI 遮擋
- 檔案命名：`003-comic-lazybag-114-ig-{01-10}.png`

**C. 備用格式**
- Facebook 分享圖：1200 x 630px（取封面格＋總結格濃縮版）
- LINE 分享圖：520 x 520px（取封面格）
