評估兩個漸進式上線方案，從社群媒體擴散效益和敘事理解度的角度分析：

方案 A：現有 URL + 鎖定未完成頁面
- 分享 /controversy-timeline 等現有 URL
- Navbar 只顯示已開放頁面，未開放顯示「即將推出」
- 優點：不用維護兩套頁面，URL 一致
- 疑慮：讀者可能困惑「為什麼有些頁面打不開」

方案 B：/share/ 獨立路由
- 分享 /share/controversy-timeline 等獨立頁面
- 與主站完全隔離，不連到未完成的頁面
- 優點：分享體驗完全可控
- 疑慮：要維護兩套頁面，URL 之後要改

需要評估：
1. 社群分享時，哪種 URL 結構更容易被理解和轉發？
2. 讀者第一次點進來的體驗哪個更好？
3. 「即將推出」的頁面是加分（建立期待感）還是扣分（感覺未完成）？
4. 對 SEO 和 OG meta（社群預覽）的影響
5. 之後正式上線時的轉換成本

## Stage Report

### 分析背景

網站現有路由結構：`/future`、`/past`、`/present`、`/opinion-lazybag`、`/controversy-timeline`、`/about`、`/preview`。社群貼文規劃為 8 篇、4 週的漸進式發佈，從 Track 3（/future 危機感）開場，依序展開三軌敘事。

---

### 方案 A 分析：現有 URL + 鎖定未完成頁面

**社群分享角度：**

- URL 乾淨直覺：`/future`、`/past` 語義清楚，在 Threads/IG 限動貼文中易於辨識
- 分享後的 URL 永遠有效——讀者收藏的連結不會過期或跳轉
- 讀者轉發時不需要解釋「這是預覽版」，URL 本身就是正式路徑
- Navbar 只顯示已開放頁面，是一種「策展」行為：等同告訴讀者「你現在該看這些」

**第一次造訪體驗：**

- 讀者從社群點入 /future，看到完整頁面內容，Navbar 顯示 1-2 個可用頁面
- 如果好奇心驅使點到鎖定頁面（透過直接輸入 URL 或搜尋引擎），會看到「即將推出」
- 頁面數量少但完整，不會有「半成品」的感覺

**「即將推出」頁面效果：**

- 對主動探索的讀者可以建立期待感：「原來還有更多內容要來」
- 但在社群分享情境中，讀者通常只看被導流的那一頁，不太會主動探索 Navbar 以外的路徑
- 風險：如果 Navbar 隱藏了未完成頁面，讀者幾乎不會遇到「即將推出」——這個功能存在但不太會被觸發

**OG meta / 社群預覽：**

- 每個頁面用正式 URL，OG meta 只需設定一次，永久有效
- 社群平台快取 OG 資料後不需更新（URL 不變）
- 分享預覽卡片上顯示的是最終域名路徑，品牌感一致

---

### 方案 B 分析：/share/ 獨立路由

**社群分享角度：**

- `/share/future` 多了一層路徑，URL 稍長但仍可接受
- 但 `/share/` 前綴傳遞了一個訊號：「這是特別分享用的版本」，可能讓讀者覺得不是「正式」內容
- 轉發時讀者可能疑惑：為什麼網址是 /share/？正式版在哪？
- 當全站上線後，舊的 /share/ 連結需要 redirect，否則所有社群貼文中的連結失效

**第一次造訪體驗：**

- 完全隔離的環境，沒有 Navbar 導向未完成頁面——體驗最乾淨
- 但也最封閉：讀者看完一頁後沒有地方可以繼續探索，除非回到社群找下一篇貼文
- 缺乏網站整體感，像是「landing page」而非「一個有規模的專案」

**OG meta / 社群預覽：**

- 需要為 /share/ 路由和正式路由各設定一套 OG meta
- 上線後若 redirect /share/ → 正式路由，社群平台快取的 OG 預覽可能不會自動更新
- Facebook 和 LINE 的 OG 快取更新需要手動清除，增加維運負擔

### 比較表

| 評估維度 | 方案 A（現有 URL + 鎖定） | 方案 B（/share/ 獨立路由） |
|---------|------------------------|------------------------|
| URL 易理解度 | ✅ 語義清楚、短、好記 | ⚠️ /share/ 前綴增加疑惑 |
| 社群轉發友善度 | ✅ 一次分享永久有效 | ❌ 上線後連結需 redirect |
| 第一次造訪體驗 | ✅ 完整頁面 + 精簡 Navbar | ✅ 乾淨但封閉，無探索空間 |
| 「即將推出」效果 | ⚠️ 可建立期待但少人觸發 | N/A（不存在此問題） |
| OG meta 維護成本 | ✅ 設定一次，永久有效 | ❌ 兩套設定 + 快取清除 |
| 轉換成本（全站上線） | ✅ 零成本，只需開放 Navbar | ❌ redirect 設定 + 快取更新 |
| 敘事弧線支撐 | ✅ 每週開放新頁面 = 連載感 | ⚠️ 頁面互相隔離，缺乏累積感 |
| 開發維護成本 | ✅ 只需條件式隱藏 Navbar 項目 | ❌ 維護兩套路由和頁面 |

### 建議：方案 A

**核心理由：社群敘事需要「累積感」，而非「隔離感」。**

從 content-plan 的 8 篇貼文結構來看，整個社群策略是一條從「危機（T3）→ 歷史（T1）→ 深度（T2）→ 整合」的敘事弧線。這條弧線的力量來自「每一週，網站多了一塊拼圖」的連載感。

方案 A 天然支撐這個敘事：
1. **第 1 週**：分享 /future，Navbar 只有「未來」一個入口——讀者聚焦危機
2. **第 2-3 週**：開放 /past，Navbar 多了「過去」——回訪讀者會發現「上次來只有一頁，現在多了一頁」，這本身就是一個故事
3. **第 4 週**：全部開放，三軌串聯——讀者看到完整全貌，對應貼文 #8 的行動呼籲

方案 B 的 /share/ 隔離頁面無法產生這種累積效果。每次點進去都是一個孤立的 landing page，讀者不會感受到「這個網站正在長大」。

**關於「即將推出」頁面的建議：**

既然 Navbar 會隱藏未開放頁面，主動遇到「即將推出」的機率很低。建議不需要特別設計「即將推出」頁面。如果有人透過直接輸入 URL 到達未開放頁面，簡單顯示一行「這個頁面即將推出，先看看已經上線的內容 →」並導回首頁即可。把精力放在已開放頁面的品質上。

**轉換成本為零：** 全站上線時只需要把 Navbar 的條件判斷移除，所有頁面自然可見。不需要 redirect、不需要清快取、不需要改 OG meta。這是決定性的技術優勢。
